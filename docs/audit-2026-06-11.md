# Codebase Audit — 2026-06-11

Full audit of wosweat (backend scraper, frontend, configs, specs). Items are checked off as they get fixed; unchecked items are open work for future sessions. Checked items below were fixed on 2026-06-11 (same session).

## Bugs

- [x] **Year rollover in date parser** (`backend/functions/utils/date-parser.ts`): dates without a year now roll to next year when >60 days in the past (`inferYear`).
- [x] **DateHeading timezone bug**: now parses with `'T00:00:00'` like the other components.
- [x] **Light theme breaks event cards**: fixed color-only via non-flipping `--overlay-*` tokens for card text and a shared `--on-accent` token for text/icons on teal fills (today pill/cell, active chip, a11y buttons, FAB icon). Verified per-theme in Cypress.
- [x] **Unstable, collision-prone event IDs**: extractors now use `eventId()` (djb2 hash of venue|date|title|url, `backend/functions/utils/event-id.ts`); `scrapeAllVenues` dedupes by id.
- [x] **Silent venue failures**: rejected scrapes are logged per venue in `scrapeAllVenues`.
- [x] **`trigger-scrape` accepts any HTTP method**: now 405 for non-POST.
- [x] **AccessibilityFab**: `saveSettings` moved out of the setState updater; dyslexia font load has catch + no duplicate FontFace registration.
- [x] **`extractDateFromText` ISO support**: `YYYY-MM-DD` is now the first pattern.
- [x] **manifest.json scaffold**: real name/colors/lang; icon points at the existing 64px favicon. *Remaining: create proper 192px/512px PWA icons — 64px is below installability requirements.*
- [ ] **Font-scale a11y setting barely works** (fix reverted, see layout-revert note): scale applies to `body` font-size, but most component sizes are rem/px → little visible effect.
- [x] **Unit test red** (jsdom `scrollIntoView`): optional-call guard + real assertions; `vitest run` green.
- [x] **Cypress e2e was scaffold**: now asserts the real app shell; verified green against Vite dev server.

## Layout revert (2026-06-11, late)

A layout regression persisted in the user's environment that could not be reproduced locally (Cypress matrix: Chromium, fixture + real data, 3 viewports, reduced-motion on/off, a11y storage variants — all healthy). At the user's request, **all layout-affecting changes were reverted to the original state**: every component CSS file is byte-identical to pre-session HEAD, `html`-based font scaling went back to `body`-based, and heading/anchor/button element conversions went back to the original divs/spans. Non-visual work was kept: aria attributes/labels, `inert`, dialog behavior, focus-visible outline, reduced-motion damping, high-contrast tokens, all backend fixes, and behavior bug fixes. Items above marked "(fix reverted)" are open again.

## Found while debugging (2026-06-11, post-audit)

- [x] **Featured card collapsed on sparse days** (pre-existing, surfaced by stale data): image-less cards had no in-flow content, so a lone featured card (grid rows unsized by siblings) collapsed to ~16px. Fixed by giving the no-image branch an in-flow placeholder that mirrors the `<img>` (100%/100%, 3/2 intrinsic stand-in) — image and image-less cards now size identically; featured CSS untouched. **Do not** fix this via `aspect-ratio` on the featured card itself: that approach regressed in Firefox (caught by the user; Chromium-only verification missed it). Verify layout changes in Firefox too.
- **Stale local Blobs data mimics bugs**: the dev Blobs sandbox persists between sessions — a months-old snapshot leaves 0–2 future events per day ("Daten vom …" hint shows the age). Re-run `npm run dev:scrape` after long gaps before suspecting scraper/UI regressions.

## Dev environment

- [x] Combined dev: `npm run dev` (root) runs `netlify dev --filter @wosweat/backend`; `backend/netlify.toml` `[dev]` boots Vite (`npm --prefix ../frontend run dev`, `targetPort` 5173) and proxies everything on 8888. ## Loose ends / unfinished features (from `docs/superpowers/specs/2026-03-27-frontend-ui-design.md`)

- **`time` + `tags` extraction pipeline**: spec'd as required backend change; EventCard already renders time badges and tag chips but no extractor populates the fields. Bäckerei discards `.event-thumb__time` and stuffs tags into the description string.
- **Reading line** (a11y drawer §8): spec'd, never built.
- **High-contrast 2px borders**: spec'd, only colors implemented.
- **Service worker / offline**: deferred in spec, still absent.
- **Performance budget exceeded**: main bundle ~261 kB gzipped vs 250 kB budget; no code splitting (Ionic + GSAP in one chunk).
- **PWA icons**: only a 64px favicon exists; installability needs 192/512px icons.
- **DST cron note**: schedule `1 22,23 * * *` + `viennaHour() !== 0` guard handles CET/CEST — intentional, don't "simplify".
- **Stale-data hint threshold** `30 * 60 * 60 * 1000` (30 h) is intentional, documented in the cache-refresh spec.
- Spec's external uptime monitor for `lastUpdated > 30 h` was never set up.
