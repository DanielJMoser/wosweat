## Found while debugging (2026-06-11, post-audit)

- [ ] **Featured card collapses on sparse days** (pre-existing, surfaced by stale data; fix reverted with the layout revert): at ≥640px, `.event-card--featured` has `aspect-ratio: unset; height: 100%` — height comes entirely from grid rows sized by sibling cards. On a day with one image-less (or dead-image) event, the card collapses to ~16px (verified in Cypress). Known fix: `aspect-ratio: 16/7` fallback — dense-day layout stays pixel-identical (verified 1/2/4-event matrix).
- **Stale local Blobs data mimics bugs**: the dev Blobs sandbox persists between sessions — a months-old snapshot leaves 0–2 future events per day ("Daten vom …" hint shows the age). Re-run `npm run dev:scrape` after long gaps before suspecting scraper/UI regressions.

## Dev environment

- [x] Combined dev: `npm run dev` (root) runs `netlify dev --filter @wosweat/backend`; `backend/netlify.toml` `[dev]` boots Vite (`npm --prefix ../frontend run dev`, `targetPort` 5173) and proxies everything on 8888. The `--filter` is required — without it the CLI's monorepo picker selects an arbitrary workspace and functions 404.
- Local Blobs store starts empty: run `npm run dev:scrape` once after `npm run dev` to populate events locally.
- `.netlify/state.json` (site link) disappeared during the picker mishap; run `netlify link` if the CLI prompts.

## Loose ends / unfinished features (from `docs/superpowers/specs/2026-03-27-frontend-ui-design.md`)

- **`time` + `tags` extraction pipeline**: spec'd as required backend change; EventCard already renders time badges and tag chips but no extractor populates the fields. Bäckerei discards `.event-thumb__time` and stuffs tags into the description string.
- **Reading line** (a11y drawer §8): spec'd, never built.
- **High-contrast 2px borders**: spec'd, only colors implemented.
- **Service worker / offline**: deferred in spec, still absent.
- **Performance budget exceeded**: main bundle ~261 kB gzipped vs 250 kB budget; no code splitting (Ionic + GSAP in one chunk).
- **PWA icons**: only a 64px favicon exists; installability needs 192/512px icons.
- **DST cron note**: schedule `1 22,23 * * *` + `viennaHour() !== 0` guard handles CET/CEST — intentional, don't "simplify".
- **Stale-data hint threshold** `30 * 60 * 60 * 1000` (30 h) is intentional, documented in the cache-refresh spec.
- Spec's external uptime monitor for `lastUpdated > 30 h` was never set up.
